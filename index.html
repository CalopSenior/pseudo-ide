Listas: Dinâmicas vs. Fixas

                

                    Lista dinâmica (m.lista(...)) — criada pela biblioteca metodos,
                    com métodos como .adicionar(), .ordenar(), etc.


                    Lista fixa (super x = [...]) — criada com colchetes e o tipo super.
                    É um array nativo JavaScript. Use quando só precisa de um conjunto fixo de valores e acesso por índice (x[0]).
                

                
// Lista DINÂMICA — usa métodos da biblioteca metodos
importar metodos como m;
super notas = m.lista(7, 8, 9);
notas.adicionar(10);
imprima(notas.tamanho());    // 4

// Lista FIXA — acesso direto por índice
super dias = ["Seg", "Ter", "Qua"];
imprima(dias[0]);            // "Seg"


                
Validação Rigorosa de Tipos

                

                    As bibliotecas nativas (mat, metodos, calculo, estatistica)
                    verificam ativamente o tipo dos argumentos. Passar um tipo errado gera um erro claro em PT-BR
                    antes da execução continuar — sem conversões silenciosas.
                


                
Erros em Português

                

                    Todos os erros de execução são traduzidos automaticamente para PT-BR com indicação da
                    [Linha X] aproximada do pseudocódigo onde ocorreram.

Parâmetros com Tipo e Restrição

                

                    Adicione o tipo e uma restrição opcional entre parênteses.
                    A IDE valida os valores em tempo de execução e lança um erro claro se forem violados.
                

                
funcao raizQuadrada(real(x >= 0)) {
    retorno raiz(x);
}
imprima(raizQuadrada(9));   // 3
imprima(raizQuadrada(-1));  // Erro: 'x' deve satisfazer: x >= 0


                
Parâmetros com Valor Padrão

                

                    Use : valor após o parêntese para definir um padrão.
                    Parâmetros sem padrão são obrigatórios.
                

                
funcao saudacao(caracter(nome), caracter(prefixo) : "Olá") {
    imprima(f"{prefixo}, {nome}!");
}

saudacao("Ana");             // Olá, Ana!
saudacao("Bruno", "Oi");    // Oi, Bruno!


                
Combinando Tudo

                
funcao potencia(real(base), inteiro(exp > 0) : 2) {
    retorno expo(base, exp);
}

imprima(potencia(3));        // 9    (exp usa padrão 2)
imprima(potencia(2, 8));    // 256
imprima(potencia(5, -1));   // Erro: 'exp' deve satisfazer: exp > 0


                

                    Sintaxe resumida:

                    tipo(nome) — obrigatório, sem restrição

                    tipo(nome operador valor) — obrigatório, com restrição (ex: >= 0, != 0)

                    tipo(nome) : padrão — opcional com valor padrão

                    tipo(nome operador valor) : padrão — opcional com restrição

Somatório e Produtório

                
Recebem uma funcao que aceita i e a aplicam para cada inteiro de min a max.

                

                    
FunçãoO que faz

                    
                        
m.somatorio(fn, min, max)Σ fn(min) + … + fn(max)

                        
m.produtorio(fn, min, max)Π fn(min) × … × fn(max)

                    
                

                
importar mat como m;

funcao quad(i) { retorno i * i; }
imprima(m.somatorio(quad, 1, 5));   // 55  (1+4+9+16+25)

funcao id(i) { retorno i; }
imprima(m.produtorio(id, 1, 5));    // 120 (5!)

Número — m.numero(valor)

                
Converte e inspeciona valores numéricos. Aceita inteiros, reais, verdadeiro/falso e strings numéricas.

                

                    
MétodoDescriçãoExemplo

                    
                        
.sinal()Retorna 1 (positivo), -1 (negativo) ou 0 (zero)m.numero(-5).sinal() → -1

                        
.int()Converte para inteiro truncando decimais. verdadeiro→1, falso→0m.numero(3.9).int() → 3

                        
.re()Retorna o valor como número realm.numero("3.14").re() → 3.14

                        
.decimal(n)Arredonda para n casas decimais, retornando um realm.numero(3.14159).decimal(2) → 3.14

                        

                            .fatorar(0)
                            Lista de pares [[base, exp], …] da fatoração em primos. Exige número inteiro — use .int() antes se necessário.
                            m.numero(12).fatorar(0) → [[2,2],[3,1]]
                        

                        

                            .fatorar(1)
                            String formatada com expoentes Unicode. Também exige inteiro.
                            m.numero(12).fatorar(1) → "2² × 3¹"
                        

                    
                

                

                    Atenção: .int() e .re() substituíram os antigos .inteiro() e .real().

                    .fatorar() agora rejeita valores reais com uma mensagem explicativa — converta antes:
                    m.numero(3.7).int().fatorar(1) → usa o inteiro 3.
                

                
importar metodos como m;

inteiro n = 360;
imprima(m.numero(n).fatorar(1));    // 2³ × 3² × 5¹
imprima(m.numero(-7).sinal());        // -1
imprima(m.numero(verdadeiro).int());  // 1
imprima(m.numero(3.9).int());         // 3 (trunca, não arredonda)
imprima(m.numero(3.14159).decimal(2));// 3.14

t.tabelaVerdade()

                

                    Gera automaticamente todas as combinações V/F e exibe a tabela colorida no console.
                    Suporta múltiplas expressões simultaneamente e exibição de sub-expressões intermediárias.
                


                
Assinatura

                
t.tabelaVerdade(expressoes, variaveis, mostrarIntermediarias)

                

                    
ParâmetroTipoDescrição

                    
                        

                            expressoes
                            caracter ou Lista
                            Uma expressão lógica como string, ou uma Lista de strings para comparar várias fórmulas lado a lado.
                        

                        

                            variaveis
                            Lista (obrigatória)
                            Lista de strings com os nomes das variáveis usadas nas expressões. Máx: 8 variáveis.
                        

                        

                            mostrarIntermediarias
                            booleano (padrão: falso)
                            Se verdadeiro, adiciona colunas para cada sub-expressão entre parênteses, das mais internas para as mais externas.
                        

                    
                


                
Operadores Suportados

                

                    
SímboloOperadorEquivale a

                    
                        
eConjunção (AND)p ∧ q

                        
ouDisjunção (OR)p ∨ q

                        
naoNegação (NOT)¬p

                        
xouDisjunção Exclusiva (XOR)p ⊕ q

                        
->Condicional (Implicação)p → q

                        
<->Bicondicional (Equivalência)p ↔ q

                    
                


                
Exemplos

                
importar tabular como t;
importar metodos como m;

// Exemplo 1 — Expressão simples
t.tabelaVerdade("p e q", m.lista("p", "q"));

// Exemplo 2 — Com intermediárias (mostra colunas de sub-expressões)
t.tabelaVerdade("(p e q) ou nao r", m.lista("p", "q", "r"), verdadeiro);

// Exemplo 3 — Múltiplas expressões (compara fórmulas lado a lado)
super exprs = m.lista("p -> q", "nao p ou q");
t.tabelaVerdade(exprs, m.lista("p", "q"));
// As duas colunas serão idênticas — demonstra que p->q ≡ ¬p∨q

// Exemplo 4 — Bicondicional e XOR
t.tabelaVerdade("p <-> q", m.lista("p", "q"));
t.separador();
t.tabelaVerdade("p xou q",  m.lista("p", "q"));


                

                    Coloração:
                    colunas de variáveis em cinza neutro;
                    intermediárias em verde/vermelho suave;
                    expressões principais em verde/vermelho intenso e negrito.
                    Células com erro de avaliação aparecem em amarelo (?).
                

            


            
            

                
Biblioteca de Análise Numérica

                
Uso: importar calculo como c; — Apoio a demonstrações de limites, derivadas e integrais.


                

                    
FunçãoDescriçãoParâmetros

                    
                        

                            c.limite(fn, ponto)
                            Calcula limx→ponto fn(x) numericamente. Lança erro se os limites laterais divergirem.
                            fn: funcao; ponto: numero
                        

                        

                            c.derivada(fn, ponto, ordem)
                            Derivada numérica de ordem n em um ponto. Ordem padrão: 1. Suporta até ordem 4.
                            fn: funcao; ponto: numero; ordem: inteiro ≥ 1
                        

                        

                            c.integral(fn, a, b)
                            Integral definida de fn entre a e b pela Regra de Simpson (n=1000). Precisão adequada para fins didáticos.
                            fn: funcao; a, b: numero
                        

                    
                


                
Exemplos

                
importar calculo como c;

// Limite de sen(x)/x quando x → 0 (resultado clássico: 1)
funcao f(x) { retorno raiz(x); }
imprima(c.limite(f, 0));           // ≈ 0 (raiz tende a 0)

// Derivada de x² em x=3 → deve ser 6
funcao quad(x) { retorno x * x; }
imprima(c.derivada(quad, 3));      // ≈ 6
imprima(c.derivada(quad, 3, 2));   // ≈ 2 (2ª derivada de x² é constante 2)

// Integral de x de 0 a 4 → área = 8
funcao linear(x) { retorno x; }
imprima(c.integral(linear, 0, 4)); // ≈ 8


                

                    Importante: os resultados são aproximações numéricas.
                    São adequados para verificar intuições matemáticas, mas não substituem cálculo analítico exato.
                    Valores muito próximos de singularidades podem apresentar imprecisão.
                

            


            
            

                
Biblioteca de Estatística e Probabilidade

                
Uso: importar estatistica como e;


                
Combinatória — exigem inteiros ≥ 0

                

                    
FunçãoFórmulaDescrição

                    
                        
e.fatorial(n)n!Fatorial de n. Máx: n=170.

                        
e.combinacao(n, k)C(n,k) = n! / (k! × (n−k)!)Combinações de n elementos tomados k a k (sem repetição, sem ordem).

                        
e.arranjo(n, k)A(n,k) = n! / (n−k)!Arranjos de n elementos tomados k a k (sem repetição, com ordem).

                    
                


                
Análise de Dados — exigem Lista não-vazia de números

                

                    
FunçãoDescrição

                    
                        
e.media(lista)Média aritmética (Σxᵢ / n)

                        
e.mediana(lista)Valor central após ordenação. Em listas de tamanho par: média dos dois centrais.

                        
e.moda(lista)Valor(es) mais frequente(s). Retorna uma Lista (pode ter múltiplas modas).

                        
e.variancia(lista)Variância populacional σ² = Σ(xᵢ − μ)² / n

                        
e.desvioPadrao(lista)Desvio padrão populacional σ = √variância

                    
                


                
Exemplo Completo

                
importar estatistica como e;
importar metodos como m;

// Combinatória
imprima(e.fatorial(5));         // 120
imprima(e.combinacao(10, 3));  // 120  (C(10,3))
imprima(e.arranjo(5, 2));      // 20   (A(5,2))

// Análise de dados
super notas = m.lista(7, 8, 9, 6, 8, 10, 7);

imprima(e.media(notas));         // 7.857...
imprima(e.mediana(notas));       // 8
imprima(e.moda(notas));          // [7, 8]
imprima(e.desvioPadrao(notas));  // ≈ 1.21


                

                    Validação rigorosa: todas as funções de análise rejeitam listas vazias
                    e listas com elementos não-numéricos, exibindo uma mensagem de erro clara com a posição
                    do elemento inválido.
